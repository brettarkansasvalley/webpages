📚 Overview

                        
The JAQ JSON Loader provides a powerful Domain-Specific Language (DSL) for querying and transforming JSON data. The DSL supports SQL-like operations including:

                        
    
                            
  • FROM - Select source data
    • 
                            
  • JOIN - Combine multiple data sources
    • 
                            
  • WHERE - Filter records with conditions
    • 
                            
  • GROUP BY - Aggregate data
    • 
                            
  • HAVING - Filter aggregated results
    • 
                            
  • SELECT - Choose output fields
    • 
                            
  • ORDER BY - Sort results
    • 
                            
  • DISTINCT - Remove duplicates
    • 
                            
  • WITH - Create subqueries/CTEs
    •

🔰 Basic Queries

                        
                        
Simple Selection

                        
Select all fields from a single source:

                        

                            

                                Example 1: Select all GMC products
                                Try It
                            

                            
{
  "from": {
    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "select": [
    {"expr": "gmc.mpn", "alias": "MPN"},
    {"expr": "gmc.title", "alias": "Product Title"},
    {"expr": "gmc.gtin", "alias": "GTIN"}
  ],
  "limit": 10
}

                        

                        
                        
Basic Filtering

                        
Use the where clause to filter records:

                        

                            

                                Example 2: Products with specific MPN prefix
                                Try It
                            

                            
{
  "from": {
    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "where": [
    {"field": "gmc.mpn", "op": "starts_with", "value": "OP"}
  ],
  "select": [
    {"expr": "gmc.mpn", "alias": "MPN"},
    {"expr": "gmc.title", "alias": "Title"}
  ],
  "limit": 10
}

                        

                        
                        
Sorting Results

                        
Use order_by to sort by any field:

                        

                            

                                Example 3: Sort by MPN ascending
                                Try It
                            

                            
{
  "from": {
    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "select": [
    {"expr": "gmc.mpn", "alias": "MPN"},
    {"expr": "gmc.title", "alias": "Title"}
  ],
  "order_by": [
    {"field": "gmc.mpn", "direction": "asc"}
  ],
  "limit": 10
}

                        

                        
                        
Pagination with OFFSET

                        
Use offset to skip records:

                        

                            

                                Example 4: Skip first 100 records
                                Try It
                            

                            
{

    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "select": [
    {"expr": "gmc.mpn", "alias": "MPN"},

  ],

  ],
  "offset": 100,
  "limit": 10
}

                        

                    

                    
                    
                    

                        
🔗 Joins

                        
                        
Joins allow you to combine data from multiple sources. Our DSL supports:

                        
    
                            
  • inner - Only matching records from both sides
    • 
                            
  • left - All records from left, matching from right
    • 
                            
  • right - All records from right, matching from left
    • 
                            
  • full - All records from both sides
    • 
                            
  • cross - Cartesian product (all combinations)
    • 
                        

                        
                        
Inner Join

                        
Match GMC products with Shopify variants by MPN/SKU:

                        

                            

                                Example 5: Join GMC with Shopify
                                Try It
                            

                            
{
  "from": {
    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "joins": [
    {
      "source": {
        "source_file": "shopify_zucker_products.json",
        "alias": "shopify"
      },
      "on": {
        "left": "gmc.mpn",
        "right": "shopify.variant.sku"
      },
      "join_type": "inner"
    }
  ],
  "select": [
    {"expr": "gmc.mpn", "alias": "MPN"},
    {"expr": "gmc.title", "alias": "GMC Title"},
    {"expr": "shopify.product.title", "alias": "Shopify Title"},
    {"expr": "shopify.variant.price", "alias": "Price"}
  ],
  "limit": 10
}

                        

                        
                        
Join with Array Field

                        
Join using array fields with [] notation:

                        

                            

                                Example 6: Join with variants array
                                Try It
                            

                            
{
  "from": {
    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "joins": [
    {
      "source": {
        "source_file": "shopify_zucker_products.json",
        "alias": "shopify"
      },
      "on": {
        "left": "gmc.mpn",
        "right": "shopify.variants[].sku"
      },
      "join_type": "inner",
      "skip_nulls": true
    }
  ],
  "select": [
    {"expr": "gmc.mpn", "alias": "MPN"},
    {"expr": "gmc.title", "alias": "GMC Title"},
    {"expr": "shopify.product.title", "alias": "Product"}
  ],
  "limit": 10
}

                        

                        
                        
Left Join

                        
Find all GMC products even without Shopify matches:

                        

                            

                                Example 7: Left join to find unmatched
                                Try It
                            

                            
{
  "from": {
    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "joins": [
    {
      "source": {
        "source_file": "shopify_zucker_products.json",
        "alias": "shopify"
      },
      "on": {
        "left": "gmc.mpn",
        "right": "shopify.variant.sku"
      },
      "join_type": "left",
      "skip_nulls": true
    }
  ],
  "where": [
    {"field": "shopify.variant.sku", "op": "is_null"}
  ],
  "select": [
    {"expr": "gmc.mpn", "alias": "Unmatched MPN"},
    {"expr": "gmc.title", "alias": "Title"}
  ],
  "limit": 10
}

                        

                        
                        
Multiple Joins

                        
Chain multiple joins together:

                        

                            

                                Example 8: GMC + Shopify + Odoo three-way join
                                Try It
                            

                            
{
  "from": {
    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "joins": [
    {
      "source": {
        "source_file": "shopify_zucker_products.json",
        "alias": "shopify"
      },
      "on": {
        "left": "gmc.mpn",
        "right": "shopify.variant.sku"
      },
      "join_type": "inner",
      "skip_nulls": true
    },
    {
      "source": {
        "source_file": "odoo_products.json",
        "alias": "odoo"
      },
      "on": {
        "left": "gmc.mpn",
        "right": "odoo.default_code"
      },
      "join_type": "left",
      "skip_nulls": true
    }
  ],
  "select": [
    {"expr": "gmc.mpn", "alias": "MPN"},
    {"expr": "gmc.title", "alias": "GMC Title"},
    {"expr": "shopify.variant.price", "alias": "Shopify Price"},
    {"expr": "odoo.list_price", "alias": "Odoo Price"}
  ],
  "limit": 10
}

AND Conditions

                        
Multiple conditions with AND logic (default):

                        

                            

                                Example 9: Multiple AND conditions
                                Try It
                            

                            
{
  "from": {
    "source_file": "shopify_zucker_products.json",
    "alias": "s"
  },
  "where": [
    {"field": "s.variant.inventoryQuantity", "op": ">", "value": 0},
    {"field": "s.variant.price", "op": ">=", "value": "10.00"},
    {"field": "s.product.vendor", "op": "=", "value": "ZUCKER"}
  ],
  "where_op": "and",
  "select": [
    {"expr": "s.variant.sku", "alias": "SKU"},
    {"expr": "s.variant.price", "alias": "Price"},
    {"expr": "s.variant.inventoryQuantity", "alias": "Stock"}
  ],
  "limit": 10
}

                        

                        
                        
OR Conditions

                        
Use OR logic for alternative conditions:

                        

                            

                                Example 10: OR conditions
                                Try It
                            

                            
{
  "from": {
    "source_file": "gmc_zucker_products.json",
    "alias": "gmc"
  },
  "where": [
    {"field": "gmc.title", "op": "contains", "value": "Ostrich"},
    {"field": "gmc.title", "op": "contains", "value": "Feather"}

  "where_op": "or",
  "select": [
    {"expr": "gmc.mpn", "alias": "MPN"},
    {"expr": "gmc.title", "alias": "Title"}
  ],
  "limit": 10
}

📈 Aggregation

                        
                        
Aggregate Functions

                        
    
                            
  • sum(field) - Sum of values
    • 
                            
  • count(field) - Count of non-null values
    • 
                            
  • avg(field) - Average value
    • 
                            
  • min(field) - Minimum value
    • 
                            
  • max(field) - Maximum value
    • 
                        

                        
                        
GROUP BY with Aggregation

                        

                            

                                Example 15: Count products by type
                                Try It
                            

                            
{
  "from": {
    "source_file": "shopify_zucker_products.json",
    "alias": "s"
  },
  "group_by": [
    {"field": "s.product.productType"}
  ],
  "select": [
    {"expr": "s.product.productType", "alias": "Product Type"},
    {"expr": "count(s.variant.sku)", "alias": "Product Count"}
  ],
  "order_by": [
    {"field": "count(s.variant.sku)", "direction": "desc"}
  ],
  "limit": 15
}

                        

                        
                        
HAVING - Filter Aggregated Results

                        

                            

                                Example 16: Filter groups with HAVING
                                Try It
                            

                            
{
  "from": {
    "source_file": "shopify_zucker_products.json",
    "alias": "s"
  },
  "group_by": [
    {"field": "s.product.productType"}
  ],
  "having": [
    {"field": "count(s.variant.sku)", "op": ">", "value": 50}
  ],
  "select": [
    {"expr": "s.product.productType", "alias": "Product Type"},
    {"expr": "count(s.variant.sku)", "alias": "Product Count"},
    {"expr": "avg(s.variant.price)", "alias": "Avg Price"}
  ],
  "order_by": [
    {"field": "count(s.variant.sku)", "direction": "desc"}
  ]
}

🔄 Subqueries (WITH Clause)

                        
                        
The WITH clause lets you define named subqueries that can be referenced in the main query. This is useful for:

                        
    
                            
  • Pre-filtering data before joining
    • 
                            
  • Breaking complex queries into steps
    • 
                            
  • Reusing the same filtered dataset multiple times
    • 
                        

                        
                        
Basic Subquery

                        

                            

                                Example 18: Filter then join
                                Try It
                            

                            
{
  "with": [
    {
      "name": "gmc_filtered",
      "query": {
        "from": {
          "source_file": "gmc_zucker_products.json",
          "alias": "g"
        },
        "where": [
          {"field": "g.title", "op": "contains", "value": "Ostrich"}
        ],
        "select": [
          {"expr": "g.mpn", "alias": "mpn"},
          {"expr": "g.title", "alias": "title"},
          {"expr": "g.gtin", "alias": "gtin"}
        ]
      }
    }
  ],
  "from_subquery": "gmc_filtered",
  "select": [
    {"expr": "gmc_filtered.mpn", "alias": "MPN"},
    {"expr": "gmc_filtered.title", "alias": "Title"}
  ],
  "limit": 10
}

                        

                        
                        
Subquery Join

                        
Join a subquery with another source:

                        

                            

                                Example 19: Subquery joined with Shopify
                                Try It
                            

                            
{
  "with": [
    {
      "name": "high_value_gmc",
      "query": {
        "from": {
          "source_file": "gmc_zucker_products.json",
          "alias": "g"
        },
        "where": [
          {"field": "g.title", "op": "contains", "value": "Premium"}
        ],
        "select": [
          {"expr": "g.mpn", "alias": "mpn"},
          {"expr": "g.title", "alias": "title"}
        ]
      }
    }
  ],
  "from_subquery": "high_value_gmc",
  "joins": [
    {
      "source": {
        "source_file": "shopify_zucker_products.json",
        "alias": "shopify"
      },
      "on": {
        "left": "high_value_gmc.mpn",
        "right": "shopify.variant.sku"
      },
      "join_type": "inner",
      "skip_nulls": true
    }
  ],
  "select": [
    {"expr": "high_value_gmc.mpn", "alias": "MPN"},
    {"expr": "high_value_gmc.title", "alias": "GMC Title"},
    {"expr": "shopify.variant.price", "alias": "Price"}
  ],
  "limit": 10
}

Some JSON files contain primitive values (strings, numbers) instead of objects. Use as_primitive to wrap them into objects:

                        
                        
Query Primitive GUID Lists

                        

                            

                                Example: Query order GUIDs from primitive values
                                Try It

    "source_file": "orders_v2_orders_20260130.json",

    "as_primitive": "guid"
  },
  "select": [
    {"expr": "o.guid", "alias": "order_guid"}
  ],
  "limit": 20
}

Extract and manipulate date/time values from ISO8601 strings:

                        
                        
Available Date Functions

                        

                            
FunctionDescriptionExample

                            
hour(field)Extract hour (0-23)hour(te.inDate)

                            
date(field)Extract date (YYYY-MM-DD)date(o.closedDate)

                            
minute(field)Extract minute (0-59)minute(te.inDate)

                            
day_of_week(field)Day of week (0=Sunday)day_of_week(o.createdDate)

                            
month(field)Month (1-12)month(o.closedDate)

                            
year(field)Yearyear(o.createdDate)

Shifts by Hour

                        

                            

                                Example: Group shifts by hour of day
                                Try It
                            

                            
{
  "from": {
    "source_file": "labor_v1_timeEntries_20260130.json",
    "alias": "te"
  },
  "where": [
    {"field": "te.inDate", "op": "is_not_null"}
  ],
  "group_by": [
    {"field": "hour(te.inDate)", "alias": "hour"}
  ],
  "select": [
    {"expr": "hour(te.inDate)", "alias": "hour"},
    {"expr": "te.guid", "alias": "shift_count", "agg": "count"}
  ],
  "order_by": [
    {"field": "hour", "direction": "asc"}
  ]
}

💥 Array Explosion with Context

                        
Traverse nested arrays while preserving parent context. Perfect for orders with multiple checks and selections:

                        
                        
Single-Level Explosion

                        

                            

                                Example: Explode order checks
                                Try It
                            

                            
{
  "from": {
    "source_file": "orders_full_20251213.json",
    "alias": "o",
    "explode_with_context": {
      "path": "checks",
      "aliases": ["c"],
      "preserve": ["o.guid", "o.businessDate"]
    }
  },
  "select": [
    {"expr": "o.guid", "alias": "order_guid"},
    {"expr": "c.guid", "alias": "check_guid"},
    {"expr": "c.amount", "alias": "amount"}
  ],
  "limit": 10
}

🔀 COALESCE Function

                        
Return the first non-null value from a list of expressions. Perfect for attribution fallback:

                        
                        
Syntax

                        

                            

                                COALESCE syntax
                            

                            
coalesce(field1, field2, 'literal', ...)

                        

                        
                        
Attribution Fallback

                        
Use payment server, fall back to order server, then default:

                        

                            

                                Example: Server attribution with fallback
                                Try It
                            

                            
{
  "from": {
    "source_file": "orders_full_20251213.json",
    "alias": "o",
    "explode_with_context": {
      "path": "checks.payments",
      "aliases": ["c", "p"]
    }
  },
  "select": [
    {"expr": "o.guid", "alias": "order_guid"},
    {"expr": "coalesce(p.server.guid, o.server.guid, 'UNKNOWN')", "alias": "attributed_server"},
    {"expr": "p.amount", "alias": "payment_amount"}
  ],
  "limit": 10
}

🌐 cURL API Reference

                        
                        
You can interact with the JAQ JSON Loader API using curl from the command line or any HTTP client. The API runs on http://localhost:3000 by default.

                        
                        
Available Endpoints

                        

                            
EndpointMethodDescription

                            
/loadPOSTLoad JSON files from the configured directory

                            
/statsGETGet database statistics

                            
/filesGETList all loaded source files

                            
/queryGETQuery objects with optional JAQ filter

                            
/query/dslPOSTExecute Query DSL (joins, filters, select)

                            
/query/dsl/with-notesPOSTExecute DSL with note attachments

                            
/exportPOSTExport query results (CSV, Excel, JSONL)

                            
/queriesGET/POSTList or save queries

                            
/schemaGETGet schema for a source file

                            
/join-analysisGETAnalyze potential joins between sources

                        

                        
                        
Load JSON Files

                        
Trigger a reload of all JSON files from the configured directory:

                        

                            

                                Load all JSON files
                                Try It
                            

                            
curl -X POST http://localhost:3000/load

                        

                        
                        
Simple Query with JAQ Filter

                        
Use the /query endpoint with JAQ filters:

                        

                            

                                Query with JAQ filter
                                Try It
                            

                            
curl -s "http://localhost:3000/query?source_file=gmc_zucker_products.json&filter=.title&limit=5" | jq .

                        

                        
                        
DSL Query (Basic)

                        
Execute a DSL query using POST with JSON body:

                        

                            

                                Basic DSL query
                                Try It
                            

                            
curl -s -X POST http://localhost:3000/query/dsl \
  -H "Content-Type: application/json" \
  -d '{
    "query": "{\"from\":{\"source_file\":\"gmc_zucker_products.json\",\"alias\":\"gmc\"},\"select\":[{\"expr\":\"gmc.mpn\",\"alias\":\"MPN\"},{\"expr\":\"gmc.title\",\"alias\":\"Title\"}],\"limit\":10}"
  }' | jq .

Join multiple sources with a DSL query:
